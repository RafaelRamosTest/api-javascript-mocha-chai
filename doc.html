API Automation Project 🚀

Documentação técnica da suíte de testes automatizados para a API ServeRest.

🛠 Stack Tecnológica

Node.js Mocha Chai Allure Supertest Faker

📋 Visão Geral

Este projeto implementa uma estratégia de testes focada em qualidade e contratos. Utilizamos validação por Schema para garantir que, caso a API sofra alterações estruturais, os testes detectem a falha imediatamente.

⚙️ Estratégia e Engenharia

  • CI/CD (Pipeline): O projeto possui um workflow no GitHub Actions que executa os testes a cada push na branch main, garantindo integração contínua e relatórios automatizados via GitHub Pages.
  • Tratamento de Massa de Dados (Data Driven): Utilizamos a biblioteca Faker para a geração de massa dinâmica, garantindo a unicidade dos dados e evitando conflitos em ambientes compartilhados.
  • Boas Práticas: Arquitetura baseada em Service Layer para abstração de chamadas de API, facilitando a manutenção e reduzindo a duplicação de código.
  • Evidências (Allure Report): A suíte gera relatórios visuais detalhados via Allure, permitindo uma análise clara de falhas e tendências para stakeholders e times de produto.

🚀 Como Rodar

# 1. Clone o repositório
git clone [seu-link-aqui]

# 2. Instale as dependências
npm install

# 3. Execute os testes
npm run ci:test

🧪 Casos de Testes

  • Fluxo POST: Cadastro de usuários com dados dinâmicos.
  • Fluxo GET: Validação de contrato via JSON Schema.
  • Fluxo PUT/DELETE: Manipulação de recursos e validação de regras de negócio.

📈 Roadmap e Evolução

  • Performance: Implementar testes de carga usando K6.
  • Notificações: Integração com Slack/Discord para reportar falhas no pipeline.
  • Segurança: Adicionar scan de vulnerabilidades básicas nas chamadas de API (OWASP ZAP).
